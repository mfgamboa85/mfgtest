Comandos GIT hub

Comando Significado Tarea Sintáxis
pwd present working directory Nos advierte en que carpeta nos encontramos trabajando pwd
mkdir make dir Crea carpeta en el directorio donde nos encontramos actualmente mkdir nuevoDirectorio
rmdir make dir Elimina directorios de la carpeta en que estamos posicionados rmdir directorio
cd change directory Nos posiciona en otro directorio de trabajo cd nombreDeCarpeta
ls list Lista de todos los archivos y carpetas en la carpeta donde estamos posicionados ls
mv move Mueve el archivo de un fichero a otro y/o lo renombra mv nombre_archivo_original.ext /posible_nueva_carpeta/cambio_nombre_archivo_opcional.ext
cp copy Copia un archivo y deja el original intacto cp notas.ext notas-copia.ext
rm remove Elimina un archivo o directorio definitivamente de nuestro entorno rm archivo_a_remover.ext
Staging Staging Nos muestra la lista de archivos a subir en el próximo commit n/a
git init git init Inicia GIT con los archivos modificados, historial, etc git init
git commit git commit Ejecuta los cambios que hicimos. Con 'git commit -m ""', agregamos un mansaje dentro de las comillas git commit / git commit -m "mensaje"
git status git status Nos muestra los archivos que se encuentran en staging o los pendientes a agregar a staging git status
git add git add Agrega un archivo al repositorio. Al ejecutar 'git add .' agrega todos los archivos pendientes al repositorio git add archivo.ext / git add .
git diff git diff Nos indica exactamente que cambios se hicieron en un archivo git diff
git checkout git checkout Deshace los cambios que vemos desde git diff. Con el punto final se deshacen los cambios de todos los archivos. Reemplazando el punto con el número de revisión, nos trasladamos a una versión anterior del proyecto, por lo que cualquier cambio aquí puede hacernos perder información. Antes de hacer cambios debemos volver a la revisión más actual con 'git checkout master' git checkout nombre_archivo.ext / git checkout . / git checkout [revision] / git checkout master
git log git log Muestra los autores de los cambios realizados en los commit. Con 'git log -p' recopilamos los detalle de la última modificación. Con 'git log -- nombre_archivo.ext' vemos las modificaciones propias de ese archivo git log nombre_archivo.ext / git log . / git log -- nombre_archivo.ext
git branch git branch Esto nos permite crear una nueva rama de trabajo en paralelo con el proyecyo actual. Se puede separar diseño, contenido, etc. Con -d eliminamos la nueva rama git branch nombre_de_la_nueva_rama / git branch -d nombre_de_la_rama
git merge git merge Une los cambios de las dos ramas desde el momento de bifuración hasta el momento git merge nombre_de_la_rama
git clone git clone Clona repositorios locales o por url git clone archivo_o_url
git remote add origin git remote add origin Descarga los repositorios a través de una url git remote add origin nombre_de_la_rama
git pull git pull Descarga ficheros a nuestra versión git pull archivo_o_url
git push git push Sube los ficheros modificados a un servidor o local git push archivo_o_url
